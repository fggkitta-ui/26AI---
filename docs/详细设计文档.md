# 详细设计文档

> 本文档为占位版本，提交比赛前需补充完整内容。整体架构与模块划分见根目录 README.md「架构」章节，此处补充各模块的详细设计要点。

## 1. 意图解析层

- 双通道设计：规则解析（快、零成本，覆盖常规问法）+ LLM 结构化抽取（覆盖长尾自然语言表达）
- 字段冲突时以 LLM 结果为准，规则结果用于合理性校验（时间范围是否落在合理区间等）

## 2. 数据源适配层

- 插件化 Adapter 设计：`BaseAdapter` 定义统一接口（login / search / fetch_detail），新增数据源无需改动主流程
- 需登录数据源：Cookie/Storage State 持久化，避免频繁触发登录风控

## 3. 清洗去重层

- 清洗：正则去除已知噪声模式（版权声明、联系方式、公众号引导等）
- 去重：标题归一化 + 发布日期 + 正文片段生成指纹哈希；进阶方案可升级为 embedding 相似度去重

## 4. 汇总生成层

- 抽取式摘要为主，避免生成式摘要引入事实性错误
- 严格保留来源链接，实现"摘要可回溯到原文"

## 5. 调度与增量推送层

- APScheduler 封装一次性任务与 cron 周期任务
- SQLite 记录已推送内容指纹，按订阅任务（用户问题）维度做增量过滤
