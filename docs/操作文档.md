# 招投标信息智能聚合工具 —— 操作文档

## 一、环境依赖安装

### 1.1 系统要求

- Python 3.10 及以上
- Windows / macOS / Linux
- 建议使用虚拟环境

### 1.2 安装步骤

```bash
# 1. 克隆仓库
git clone <本仓库地址>
cd tender-info-aggregator

# 2. 创建虚拟环境（推荐）
python -m venv .venv

# Windows 激活
.venv\Scripts\activate

# macOS/Linux 激活
source .venv/bin/activate

# 3. 安装 Python 依赖
pip install -r requirements.txt

# 4. 安装 Playwright 浏览器驱动（仅在使用 Playwright 抓取/登录时需要）
playwright install chromium
```

### 1.3 配置 LLM API Key（可选，LLM 意图解析需要）

```bash
# Windows
set ANTHROPIC_API_KEY=sk-ant-api03-xxxxx

# macOS/Linux
export ANTHROPIC_API_KEY=sk-ant-api03-xxxxx
```

### 1.4 配置登录网站的账号密码（可选，需要登录态的数据源）

方式一：环境变量
```bash
export CEB_USERNAME=your_username
export CEB_PASSWORD=your_password
```

方式二：创建 `config/secrets.yaml`（已加入 .gitignore）
```yaml
sources:
  cebpubservice:
    username: your_username
    password: your_password
```

## 二、运行方式

本系统提供三种交互方式：CLI、Web UI、API。

### 2.1 CLI 模式（命令行）

**基本用法**：
```bash
python -m src.main "你的自然语言问题"
```

**示例 1 —— 立即执行**：
```bash
python -m src.main "最近1个月的安徽省区域内的服务器招标信息都有哪些"
```

**示例 2 —— 定时执行**：
```bash
python -m src.main "最近3个月的上海区域内的充电桩招标信息都有哪些，请汇总后每天9:00发送给我"
```

**示例 3 —— 演示模式**（无需网络，使用内置演示数据验证全流程）：
```bash
python -m src.main --demo "最近3个月的上海区域内的充电桩招标信息都有哪些"
```

**示例 4 —— 查看已注册任务**：
```bash
python -m src.main --list-jobs
```

**示例 5 —— 详细日志模式**：
```bash
python -m src.main -v "最近1个月的安徽省区域内的服务器招标信息都有哪些"
```

### 2.2 Web UI 模式

启动 Web 界面：
```bash
python -m src.main --web
```

然后在浏览器中打开 `http://127.0.0.1:8080`。

Web UI 功能：
- **查询招投标信息**：输入自然语言问题，点击"开始聚合"
- **仅解析意图**：查看系统对输入的理解，不实际抓取
- **演示模式**：勾选"演示模式"使用内置数据
- **历史报告**：查看和下载之前生成的 Word 文件
- **快速示例**：点击预设示例自动填入

自定义端口：
```bash
python -m src.main --web --host 0.0.0.0 --port 5000
```

### 2.3 程序化调用（Python API）

```python
from src.intent.rule_parser import parse_intent
from src.main import run_pipeline

# 仅解析意图
intent = parse_intent("最近1个月的安徽省服务器招标信息")

# 执行完整流程
output_path = run_pipeline("最近1个月的安徽省服务器招标信息", demo_mode=True)
print(f"报告已生成: {output_path}")
```

## 五、输出物说明

### 5.1 Word 文件命名规则

`{用户原始问题}_{YYYYMMDDHHmm}.docx`

实例：
- `最近3个月的上海区域内的充电桩招标信息都有哪些_202607171424.docx`
- `最近1个月的安徽省区域内的服务器招标信息都有哪些_202607171430.docx`

### 5.2 Word 文件内容结构

1. **标题**：用户原始问题
2. **元信息**：生成时间（斜体）、检索条件摘要
3. **增量说明**：定时任务增量推送时标注
4. **信息列表**：每条信息包含
   - 标题（Heading 2）
   - 发布时间
   - 来源链接（原文 URL）
   - 核心内容（抽取式摘要，与原文事实一致）
   - 附件链接（如有）

### 5.3 运行时数据

- `data/pushed_state.db`：SQLite 数据库，存储已推送内容的指纹
- `data/session_*.json`：登录态缓存文件（如有配置登录源）
- `outputs/*.docx`：生成的 Word 报告文件

## 六、常见问题

**Q: 提示 "No module named 'src'"？**
A: 请在 `tender-info-aggregator/` 目录下执行命令，不要在 `src/` 目录下执行。

**Q: 抓取返回空结果？**
A: 三个原因可能：(1) 目标网站暂时不可达 (2) 该时间段/关键词确实没有匹配公告 (3) 网络防火墙限制。使用 `--demo` 模式可以验证全流程。

**Q: Playwright 报错？**
A: 执行 `playwright install chromium` 安装浏览器驱动。

**Q: 如何更换数据源？**
A: 编辑 `src/main.py` 中的 `ADAPTERS` 列表，添加/移除适配器实例。新增适配器只需继承 `BaseAdapter` 并实现三个方法。

**Q: 如何添加新的同义词？**
A: 编辑 `src/pipeline/cleaner.py` 中的 `SYNONYM_MAP` 字典。
