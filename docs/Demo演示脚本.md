# 招投标信息智能聚合工具 —— Demo 演示脚本

## 录制准备

- [ ] 确认终端已激活虚拟环境
- [ ] 确认 `pip install -r requirements.txt` 已完成
- [ ] 清空 outputs/ 目录（可选，便于展示新文件生成）
- [ ] 准备好录屏软件（OBS / Screen Studio / 系统自带录屏）

## 分镜 1：开场介绍 (30 秒)

**画面**：终端 + README.md

**旁白**：
> 这是一个面向企业场景的招投标信息智能聚合工具。你只需要用自然语言描述需求，
> 系统就会自动从多个招投标网站抓取信息，完成清洗、去重、汇总，最终生成
> 一份专业的 Word 报告。如果指定了发送频率，还会定时增量推送。

**操作**：
```bash
cd tender-info-aggregator
cat README.md | head -30
```

## 分镜 2：意图解析演示 (1 分钟)

**画面**：终端，运行意图解析器

**旁白**：
> 我们先看意图解析是怎么工作的。用户用自然语言输入问题，系统自动识别出
> 关键词、区域、时间范围和调度方式。注意这里用的是通用规则，不是硬编码。

**操作**：
```bash
python -m src.intent.rule_parser
```

**重点展示**：
- 6 个不同的自然语言问法（含 2 个题目没给的新问法）
- 每个都正确解析出 keyword / region / time_range / schedule
- "每周一9:00"被正确转换为 cron 表达式 `0 9 * * 1`

## 分镜 3：完整流程——立即执行 (1.5 分钟)

**画面**：终端，执行完整流程

**旁白**：
> 现在我们来跑一个完整的流程。输入"最近1个月的安徽省区域内的服务器招标信息"，
> 系统会自动解析意图、从 3 个数据源抓取信息、清洗去重、汇总生成 Word 报告。

**操作**：
```bash
# 清空之前的输出（可选）
rm -f outputs/*.docx

# 执行完整的聚合流程（演示模式）
python -m src.main --demo "最近1个月的安徽省区域内的服务器招标信息都有哪些"
```

**重点展示**：
- 终端打印的意图解析结果（关键词/区域/时间/调度）
- 各数据源的抓取日志
- 清洗/去重/增量过滤的统计数字
- "✅ 完成！已生成文档"的最终输出

**打开生成的 Word 文件**：
- 展示标题、检索条件、信息列表
- 每条信息包含：标题、发布时间、来源链接、核心内容、附件链接

## 分镜 6：定时 + 增量推送 (2 分钟)

**画面**：终端，执行定时任务模式

**旁白**：
> 这是赛题要求的核心场景——用户说"每天9:00发送给我"。
> 系统会注册一个周期任务，并且每次只推送新增内容，不重复推送。

**第一次执行**：
```bash
python -m src.main --demo "最近3个月的上海区域内的充电桩招标信息都有哪些，请汇总后每天9:00发送给我"
```

按 `Ctrl+C` 退出后：

```bash
# 查看已推送记录
python -m src.main --list-jobs
```

**第二次执行（同一个查询）**：
```bash
python -m src.main --demo "最近3个月的上海区域内的充电桩招标信息都有哪些，请汇总后每天9:00发送给我"
```

**重点展示**：
- 第一次执行：生成了包含多条信息的报告
- `--list-jobs` 显示了已推送的记录
- 第二次执行：same query → 文档中显示"本次未检索到符合条件的新增招投标信息。"（因为演示数据已经全部推送过了）
- 文档中标注了"说明：本次为定时任务的增量推送，仅包含本次新增内容。"

4. **切换到"历史报告"页**：展示已生成的文件列表

> 7. CLI + Web UI 双交互方式
